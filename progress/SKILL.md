# Progress Skill

追踪项目开发进度，实现跨设备的工作现场保存与恢复。

## 全局规则

1. **项目根目录检测**：从当前目录向上查找，直到找到包含 `.git` 或 `PROGRESS.md` 的目录作为项目根。
2. **文件路径**：所有操作均针对项目根目录下的 `PROGRESS.md`。
3. **语言跟随用户**：分析历史 commit 和用户输入，保持中英文一致性。
4. **若 PROGRESS.md 不存在**：
   - 询问用户："检测到项目尚未追踪进度，是否初始化 PROGRESS.md？"
   - 用户确认后，直接在项目根目录写入以下标准结构：

```
# Progress

> 最后更新: {CURRENT_DATE}

## 🎯 当前聚焦
<!-- 正在处理的核心任务，建议不超过 2 条 -->

## 📥 待办队列
<!-- 下一步计划做的事 -->

## ✅ 最近完成
<!-- 仅保留最近 3-5 条，避免文件无限膨胀 -->

## 🧱 阻塞与问题
<!-- 记录卡点，方便回溯 -->

## 🧠 上下文备忘
<!-- 关键决策、API 片段、调研结论 -->

## ⚡ 快速恢复
- `git pull`
-
```
